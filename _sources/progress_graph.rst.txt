Aurora Progress Graph
=====================

This page tracks Aurora production progress across all tracked Aurora grid
configs. The graph is regenerated during docs deployment and reports three
completion channels per grid:

* **Stage 1 Climate**: converged climate cache groups.
* **Stage 2 Spectra**: generated NetCDF spectra.
* **QC Evaluated**: runs with QC status rows in the report set.

Aurora Route Flowchart
----------------------

The workflow below mirrors the ReflectX-style route selection used in Aurora's
RoadRunner/PICASO decision flow.

.. mermaid::

   flowchart TD
       startNode["Start: SystemParams (Teff, logg, Rp, a, phase)"] --> routePick{"Pick run route"}

       routePick -->|"Route A (recommended)"| routeA["thermal_source = picaso<br/>atmosphere_source = picaso"]
       routePick -->|"Route B"| routeB["thermal_source = egp<br/>atmosphere_source = picaso"]
       routePick -->|"Route C"| routeC["thermal_source = egp<br/>atmosphere_source = slgrid"]

       routeA --> aAtm["PICASO atmosphere: PT + chemistry + clouds"]
       aAtm --> aRefl["Reflected light at requested phase"]
       aAtm --> aTherm["Thermal spectrum from PICASO"]

       routeB --> bAtm["PICASO atmosphere: PT + chemistry + clouds"]
       bAtm --> bRefl["Reflected light at requested phase"]
       routeB --> bTherm["Thermal from EGP IRflux file"]

       routeC --> cAtm["SLGRID PT/cloud files loaded into PICASO"]
       cAtm --> cRefl["Reflected light at requested phase"]
       routeC --> cTherm["Thermal from EGP IRflux file"]

       aRefl --> metrics["Band metrics: f_reflect, fluxes, decision"]
       aTherm --> metrics
       bRefl --> metrics
       bTherm --> metrics
       cRefl --> metrics
       cTherm --> metrics
